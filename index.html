01 MikroTikDasar Jaringan Komputer4 JP • Kelas X ＋

Penjelasan

Jaringan komputer menghubungkan dua atau lebih perangkat agar dapat bertukar data dan memakai sumber daya bersama. LAN mencakup area kecil seperti laboratorium, sedangkan WAN menghubungkan lokasi yang berjauhan melalui jaringan operator atau internet.

Konsep inti

  • Switch meneruskan data di dalam satu LAN berdasarkan alamat MAC.
  • Router menghubungkan jaringan yang berbeda dan memilih jalur paket berdasarkan alamat IP.
  • Access point menyediakan akses nirkabel, sedangkan bandwidth adalah kapasitas jalur dan latency adalah waktu tunda.

Langkah belajar dan praktik

  1. Identifikasi fungsi perangkat yang tersedia di laboratorium.
  2. Gambar topologi star sederhana berisi router, switch, access point, dan klien.
  3. Hubungkan perangkat lalu uji koneksi antarklien dengan ping.
Contoh atau catatan penting: Contoh: komputer laboratorium terhubung ke switch, kemudian switch terhubung ke router untuk mengakses internet.
Target kompetensi: Memahami LAN, WAN, internet, topologi, media transmisi, bandwidth, latency, dan perangkat jaringan.

02 MikroTikModel OSI dan TCP/IP4 JP • Kelas X ＋

Penjelasan

Model OSI dan TCP/IP membantu menjelaskan perjalanan data dari aplikasi pengirim sampai aplikasi penerima. Setiap lapisan memiliki tugas tertentu, sehingga gangguan jaringan dapat dicari secara sistematis dari lapisan fisik hingga aplikasi.

Konsep inti

  • Enkapsulasi menambahkan header saat data turun melalui lapisan; decapsulation melepaskannya di sisi penerima.
  • TCP mengutamakan keandalan, sedangkan UDP mengutamakan kecepatan dan overhead rendah.
  • ARP mencari alamat MAC dari alamat IPv4, sedangkan ICMP dipakai antara lain oleh ping.

Langkah belajar dan praktik

  1. Urutkan tujuh lapisan OSI beserta fungsi utamanya.
  2. Amati proses membuka situs dan tentukan protokol yang bekerja.
  3. Kelompokkan contoh gangguan berdasarkan lapisan OSI.
Contoh atau catatan penting: Contoh: kabel putus termasuk masalah layer fisik, sedangkan salah alamat IP termasuk masalah layer jaringan.
Target kompetensi: Menganalisis proses komunikasi data berdasarkan lapisan protokol.

03 MikroTikIPv4 dan Subnetting Dasar6 JP • Kelas X ＋

Penjelasan

Alamat IPv4 terdiri dari 32 bit dan ditulis sebagai empat bilangan desimal. Subnet mask atau prefix CIDR menentukan bagian network dan host, sehingga kita dapat menghitung network ID, broadcast, serta jumlah alamat yang dapat dipakai.

Konsep inti

  • Alamat private dipakai di jaringan lokal dan tidak dirutekan langsung di internet.
  • Prefix yang lebih besar menghasilkan subnet lebih kecil dan jumlah host lebih sedikit.
  • Network ID menandai jaringan, broadcast digunakan untuk seluruh host dalam subnet, dan alamat di antaranya dipakai host.

Langkah belajar dan praktik

  1. Ubah subnet mask desimal menjadi prefix CIDR.
  2. Hitung network, broadcast, dan rentang host untuk /24 hingga /30.
  3. Susun tabel IP untuk dua atau tiga segmen jaringan.
Contoh atau catatan penting: Contoh: 192.168.10.0/24 memiliki network 192.168.10.0, broadcast 192.168.10.255, dan 254 host yang dapat dipakai.
Target kompetensi: Menentukan network ID, host ID, subnet mask, broadcast, dan rentang host.

04 MikroTikPengenalan MikroTik RouterOS4 JP • Kelas X ＋

Penjelasan

Contoh atau catatan penting: Contoh: CHR dapat dipasang di VirtualBox untuk menyimulasikan beberapa router dalam satu komputer.
Target kompetensi: Mengenal produk MikroTik, lisensi, RouterBOARD, CHR, dan metode akses.

06 MikroTikIP Address, Route, dan DNS6 JP • Kelas X ＋

Penjelasan

Router memerlukan alamat IP pada setiap jaringan yang terhubung. Default route mengarahkan paket ke gateway internet, sedangkan DNS menerjemahkan nama domain menjadi alamat IP agar pengguna tidak perlu mengingat angka.

Konsep inti

  • Alamat IP harus dipasang pada interface yang benar dan tidak boleh tumpang tindih.
  • Route 0.0.0.0/0 digunakan sebagai jalur untuk tujuan yang tidak memiliki route lebih spesifik.
  • Allow Remote Requests membuat router dapat melayani permintaan DNS klien, tetapi harus dibatasi dari jaringan tepercaya.

Langkah belajar dan praktik

  1. Pasang IP pada interface LAN dan WAN sesuai tabel addressing.
  2. Tambahkan default route menuju gateway penyedia layanan.
  3. Uji berurutan: ping gateway, ping IP internet, lalu ping nama domain.
Contoh atau catatan penting: Jika ping 8.8.8.8 berhasil tetapi ping nama domain gagal, pemeriksaan diarahkan ke konfigurasi DNS.
Target kompetensi: Membangun konektivitas dasar antara LAN, router, dan internet.

07 MikroTikDHCP Server dan DHCP Client5 JP • Kelas X ＋

Penjelasan

DHCP memberikan alamat IP, gateway, dan DNS secara otomatis kepada klien. Pada sisi WAN, DHCP Client dapat menerima konfigurasi dari modem atau ISP; pada sisi LAN, DHCP Server membagikan konfigurasi kepada komputer pengguna.

Konsep inti

  • Pool berisi rentang alamat yang boleh dipinjamkan.
  • Network DHCP menentukan gateway, subnet mask, dan DNS untuk klien.
  • Lease mencatat peminjaman alamat; static lease mengikat perangkat tertentu pada alamat yang tetap.

Langkah belajar dan praktik

  1. Aktifkan DHCP Client pada interface WAN bila diperlukan.
  2. Buat DHCP Server pada interface LAN dengan pool yang tidak memakai alamat router.
  3. Hubungkan klien, periksa lease, lalu ubah satu lease menjadi static.
Contoh atau catatan penting: Contoh: router memakai 192.168.10.1, sedangkan pool klien 192.168.10.10–192.168.10.200.
Target kompetensi: Mendistribusikan alamat IP otomatis pada jaringan lokal.

08 MikroTikNAT Dasar4 JP • Kelas X ＋

Penjelasan

NAT menerjemahkan alamat private menjadi alamat yang dapat digunakan saat mengakses jaringan luar. Pada koneksi internet dengan alamat WAN yang dapat berubah, action masquerade umum dipakai pada chain srcnat.

Konsep inti

  • Srcnat bekerja pada trafik yang keluar dari jaringan lokal.
  • Masquerade menggunakan alamat IP pada interface keluar secara otomatis.
  • NAT bukan pengganti firewall; aturan filter tetap dibutuhkan untuk keamanan.

Langkah belajar dan praktik

  1. Pastikan routing dan DNS sudah benar sebelum membuat NAT.
  2. Buat rule srcnat dengan out-interface WAN dan action masquerade.
  3. Uji internet dari klien serta periksa counter rule NAT.
Contoh atau catatan penting: Tanpa NAT, alamat private klien biasanya tidak dapat menerima balasan langsung dari internet.
Target kompetensi: Menghubungkan jaringan private ke internet menggunakan source NAT.

09 MikroTikFirewall Dasar6 JP • Kelas X ＋

Penjelasan

Firewall memeriksa paket dan menentukan apakah paket diterima, ditolak, atau diteruskan. Rule dibaca dari atas ke bawah, sehingga urutan aturan sangat menentukan hasil akhir.

Konsep inti

  • Chain input melindungi router, forward mengatur trafik melewati router, dan output menangani trafik yang dibuat router.
  • Connection state established dan related biasanya diizinkan agar koneksi yang sah tetap berjalan.
  • Drop membuang paket tanpa balasan, sedangkan reject memberi respons penolakan.

Langkah belajar dan praktik

  1. Izinkan established dan related pada chain input.
  2. Izinkan manajemen hanya dari jaringan administrator.
  3. Tambahkan rule drop terakhir, lalu uji agar tidak mengunci akses sendiri.
Contoh atau catatan penting: Gunakan Safe Mode saat mengubah firewall untuk mengurangi risiko kehilangan akses ke router.
Target kompetensi: Memahami packet flow dan membuat filter rule sederhana.

10 Fiber OptikPengenalan Fiber Optik4 JP • Kelas X ＋

Penjelasan

Fiber optik mengirim data sebagai cahaya melalui core. Cahaya tetap berada di dalam serat karena perbedaan indeks bias antara core dan cladding, sehingga jarak transmisi dapat jauh dengan redaman rendah.

Konsep inti

  • Singlemode memiliki core kecil dan cocok untuk jarak jauh; multimode cocok untuk jarak lebih pendek.
  • Coating dan jacket melindungi serat dari kelembapan, goresan, serta tekanan mekanis.
  • Sumber laser umum pada singlemode, sedangkan LED atau VCSEL banyak digunakan pada multimode.

Langkah belajar dan praktik

  1. Amati penampang dan lapisan kabel fiber menggunakan contoh fisik.
  2. Bandingkan karakteristik singlemode dan multimode.
  3. Identifikasi kelebihan fiber dibanding kabel tembaga.
Contoh atau catatan penting: Fiber tahan gangguan elektromagnetik, tetapi lebih sensitif terhadap tekukan dan kebersihan konektor.
Target kompetensi: Memahami prinsip rambatan cahaya dan keunggulan fiber dibanding tembaga.

11 Fiber OptikJenis Kabel dan Kode Warna5 JP • Kelas X ＋

Penjelasan

Kabel fiber dibuat dalam beberapa konstruksi agar sesuai dengan lingkungan pemasangan. Kode warna memudahkan teknisi mengenali tube dan core sehingga penyambungan tidak tertukar.

Konsep inti

  • Kabel aerial dirancang untuk pemasangan udara, duct untuk saluran, dan indoor untuk lingkungan bangunan.
  • Loose tube memberi ruang serat bergerak di dalam tube, sedangkan tight-buffered memiliki pelindung yang lebih rapat.
  • Urutan warna core harus diikuti secara konsisten pada kedua ujung link.

Langkah belajar dan praktik

  1. Kelompokkan sampel kabel berdasarkan metode instalasinya.
  2. Hafalkan dan praktikkan urutan warna 12 core.
  3. Buat label kabel yang memuat asal, tujuan, kapasitas, dan nomor core.
Contoh atau catatan penting: Kesalahan membaca warna core dapat menyebabkan pelanggan tersambung ke jalur yang salah.
Target kompetensi: Mengidentifikasi konstruksi kabel, jumlah core, dan standar warna.

12 Fiber OptikKonektor dan Perangkat Pasif5 JP • Kelas X ＋

Penjelasan

Konektor menyambungkan fiber ke perangkat atau adaptor tanpa penyambungan permanen. Perangkat pasif seperti splitter, ODF, ODC, dan ODP membagi serta mengelola jalur tanpa memerlukan catu daya.

Konsep inti

  • UPC biasanya berwarna biru, sedangkan APC berwarna hijau dan memiliki ujung miring untuk mengurangi pantulan.
  • Pigtail memiliki konektor pada satu ujung untuk disambung ke kabel, sedangkan patch cord berkonektor pada kedua ujung.
  • PLC splitter membagi daya optik ke beberapa keluaran dan menambah loss sesuai rasio pembagian.

Langkah belajar dan praktik

  1. Identifikasi konektor SC, LC, FC, dan ST.
  2. Pasangkan adaptor, pigtail, dan patch cord tanpa menyentuh ferrule.
  3. Gambar alur ODF–ODC–ODP–pelanggan pada jaringan FTTH.
Contoh atau catatan penting: Konektor APC tidak boleh dipasangkan dengan UPC karena bentuk poles berbeda dan dapat meningkatkan loss.
Target kompetensi: Mengenal SC, LC, FC, ST, adaptor, pigtail, patch cord, splitter, ODF, ODC, dan ODP.

13 Fiber OptikK3 dan Perawatan Alat4 JP • Kelas X ＋

Penjelasan

Pekerjaan fiber melibatkan laser yang tidak terlihat dan serpihan kaca sangat kecil. K3 wajib diterapkan agar teknisi, pengguna, dan alat tetap aman selama pemotongan, penyambungan, serta pengukuran.

Konsep inti

Langkah belajar dan praktik

  2. Bersihkan konektor memakai alat dan cairan yang sesuai.
  3. Periksa kondisi blade cleaver dan elektroda splicer sebelum praktik.
Contoh atau catatan penting: Kotoran mikroskopis pada ferrule dapat menyebabkan redaman besar dan merusak permukaan konektor.
Target kompetensi: Menerapkan keselamatan kerja saat menangani kabel dan alat optik.

01 MikroTikBridge, VLAN, dan STP7 JP • Kelas XI ＋

Penjelasan

Bridge menggabungkan beberapa interface pada layer 2, sedangkan VLAN memisahkan satu infrastruktur fisik menjadi beberapa jaringan logis. STP atau RSTP mencegah loop yang dapat menyebabkan broadcast storm.

Konsep inti

  • Frame tagged membawa VLAN ID, sedangkan frame untagged tidak.
  • PVID menentukan VLAN untuk frame untagged yang masuk pada sebuah port.
  • Bridge VLAN Filtering mengatur VLAN mana yang boleh lewat pada port trunk dan access.

Langkah belajar dan praktik

  1. Buat bridge dan masukkan port yang diperlukan.
  2. Konfigurasikan VLAN access dan trunk dengan PVID serta tabel VLAN.
  3. Uji isolasi antarkelompok dan simulasi loop untuk mengamati RSTP.
Contoh atau catatan penting: Contoh: VLAN 10 untuk guru dan VLAN 20 untuk siswa dapat melewati satu kabel trunk menuju switch lain.
Target kompetensi: Membangun segmentasi layer 2 serta mencegah loop jaringan.

02 MikroTikRouting Statis dan Policy Route Dasar6 JP • Kelas XI ＋

Penjelasan

Konsep inti

  • Distance menentukan prioritas ketika beberapa route menuju tujuan yang sama tersedia.

Langkah belajar dan praktik

  1. Tambahkan static route antardua atau tiga router.
  2. Ubah distance dan amati route aktif.
  3. Buat kebijakan sederhana agar satu subnet memakai gateway tertentu.
Contoh atau catatan penting: Policy route dapat mengarahkan jaringan laboratorium ke ISP kedua sementara jaringan kantor memakai ISP utama.
Target kompetensi: Menghubungkan beberapa jaringan serta menentukan jalur berdasarkan kebijakan.

03 MikroTikRouting Dinamis OSPF Dasar8 JP • Kelas XI ＋

Penjelasan

OSPF adalah protokol link-state yang bertukar informasi jaringan secara otomatis. Router membentuk adjacency, menyusun database topologi, lalu menghitung jalur terbaik berdasarkan cost.

Konsep inti

  • Area backbone menggunakan area 0 dan menjadi pusat hubungan antararea.
  • Network advertisement menentukan prefix yang diumumkan ke tetangga.
  • Cost lebih kecil biasanya lebih dipilih; kesalahan MTU, area, atau autentikasi dapat menggagalkan adjacency.

Langkah belajar dan praktik

  1. Bangun topologi minimal tiga router.
  2. Aktifkan OSPF dan iklankan jaringan yang diperlukan.
  3. Putuskan satu link, amati perubahan route, lalu analisis log tetangga.
Contoh atau catatan penting: OSPF memperbarui jalur otomatis ketika link utama terputus dan jalur alternatif tersedia.
Target kompetensi: Menerapkan pertukaran route otomatis pada beberapa router.

04 MikroTikWireless Dasar6 JP • Kelas XI ＋

Penjelasan

Jaringan wireless menggunakan gelombang radio sehingga kualitasnya dipengaruhi frekuensi, interferensi, jarak, penghalang, dan kemampuan perangkat. Konfigurasi yang baik menyeimbangkan cakupan, kapasitas, dan keamanan.

Konsep inti

Langkah belajar dan praktik

  1. Konfigurasikan satu perangkat sebagai AP dan satu sebagai station.
  2. Pilih frekuensi yang sesuai berdasarkan hasil pemindaian.
  3. Uji sinyal dan throughput pada beberapa jarak atau posisi.
Contoh atau catatan penting: Sinyal kuat tidak selalu berarti koneksi bagus jika noise tinggi atau channel sangat padat.
Target kompetensi: Mengonfigurasi access point dan station dengan keamanan yang tepat.

07 MikroTikManajemen Bandwidth7 JP • Kelas XI ＋

Penjelasan

Manajemen bandwidth menjaga agar satu pengguna tidak menghabiskan seluruh kapasitas jaringan. Simple Queue mudah digunakan untuk target tertentu, sedangkan PCQ membagi bandwidth secara lebih merata ke banyak pengguna.

Konsep inti

Penjelasan

Quality Assurance memastikan hasil instalasi memenuhi desain dan standar penerimaan sebelum diserahterimakan. Pemeriksaan meliputi kebersihan konektor, insertion loss, trace OTDR, label, kerapian, serta kelengkapan bukti.

Konsep inti

  • Inspection connector dilakukan sebelum koneksi untuk mencegah kontaminasi silang.
  • Insertion loss test menilai performa end-to-end, sedangkan OTDR menunjukkan lokasi event.
  • Berita acara mencatat alat, kalibrasi, metode, hasil, penyimpangan, dan persetujuan.

Langkah belajar dan praktik

  1. Periksa visual dan kebersihan seluruh konektor.
  2. Lakukan pengukuran power meter serta OTDR sesuai metode yang ditetapkan.
  3. Bandingkan hasil dengan batas, perbaiki kegagalan, lalu susun laporan penerimaan.
Contoh atau catatan penting: Link tidak dinyatakan lulus hanya karena koneksi menyala; nilai loss dan dokumentasinya harus memenuhi kriteria.
Target kompetensi: Menetapkan batas penerimaan hasil instalasi dan pengujian.

15 ProjectProyek Akhir: Mini ISP/FTTH Terpadu16 JP • Kelas XII ＋

Penjelasan

Proyek akhir menggabungkan perencanaan layanan, routing, keamanan, QoS, monitoring, dan desain FTTH. Tim bekerja seperti penyedia layanan kecil: menganalisis kebutuhan, membuat desain, mengimplementasikan, menguji, dan menyerahkan dokumentasi.

Konsep inti

  • HLD menjelaskan arsitektur umum, sedangkan LLD memuat detail port, IP, VLAN, route, dan konfigurasi.
  • Desain fiber harus memiliki rute, core assignment, splitter, link budget, dan kapasitas cadangan.
  • Acceptance test menguji konektivitas, performa, keamanan, failover, monitoring, serta kualitas optik.

Langkah belajar dan praktik

  1. Susun kebutuhan, SLA sederhana, HLD, LLD, BoM, dan jadwal kerja.
  2. Bangun layanan pelanggan beserta autentikasi, routing, firewall, QoS, dan monitoring.
  3. Uji skenario normal serta gangguan, lalu serahkan as-built, konfigurasi, hasil ukur, dan presentasi.
Contoh atau catatan penting: Penilaian mencakup ketepatan desain, fungsi layanan, keselamatan kerja, kualitas pengujian, dokumentasi, dan kemampuan menjelaskan keputusan teknis.
Target kompetensi: Merancang, mengimplementasikan, menguji, dan mempresentasikan jaringan akses berbasis MikroTik dan fiber optik.

Standar Praktik Laboratorium

Praktik harus dilakukan bertahap, terdokumentasi, dan mengikuti prosedur keselamatan kerja.

Alur Praktikum MikroTik

  1. Identifikasi kebutuhan dan gambar topologi logis/fisik.
  2. Reset konfigurasi sesuai instruksi, ubah identity, dan buat akun administrator baru.
  3. Konfigurasi interface, IP address, DNS, DHCP, routing, NAT, dan firewall.
  4. Uji dengan ping, traceroute, torch, log, dan packet sniffer.
  5. Simpan backup, export konfigurasi, lalu tulis laporan hasil pengujian.

Alur Praktikum Fiber Optik

  1. Gunakan APD dan amankan serpihan serat dalam wadah khusus.
  2. Identifikasi jenis kabel, core, konektor, splitter, ODF, ODC, dan ODP.
  3. Lakukan stripping, cleaving, fusion splicing, dan pemasangan protection sleeve.
  4. Ukur daya dengan optical power meter dan telusuri gangguan memakai OTDR.
  5. Dokumentasikan nilai redaman, loss event, panjang link, dan hasil akhir.
Keselamatan: jangan melihat langsung ke ujung kabel fiber, jangan menyentuh serpihan serat, matikan sumber optik saat pemeriksaan, dan jaga konektor tetap bersih.
